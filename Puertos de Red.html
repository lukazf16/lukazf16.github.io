Puertos de Red

¿Qué es un puerto de red?

Es una interfaz virtual en un dispositivo, como un ordenador o un servidor, este permite la comunicación y intercambio de datos a través de una red. Estos puertos no son fisicos, sino que son constructos lógicos identificados por números de puerto, estos son únicos.

Tipos de puertos de red

Existen tres tipos principales de puertos de red:

  • Puertos Bien Conocidos
  • Puertos Registrados
  • Puertos Dinámicos o Privados

Puertos Bien Conocidos

Estos puertos son del 0 al 1023, son utilizados por protocolos y aplicaciones.

  • Por ejemplo: El puerto 80 es utilizado por el protocolo HTTP, lo que facilita la navegación web.

Puertos Registrados

Estos puertos tienen un rango del 1024 al 49151, son menos conocidos, pero todavia son usados para cosas específicas como IANA(Internet Assigned Numbers Authority).

Puertos Dinámicos o Privados

Estos puertos van del 49152 al 65535, son asignados de forma dinámica para las aplicaciones del cliente cuendo se es necesario, así facilitando múltiples sesiones de comunicación simultáneas.

TCP Y UDP

Los puertos de red trabajan en conjunto con los protocolos de la capa de transporte del modelo OSI, principalmente con TCP (Transmission Control Protocol) y UDP (User Datagram Protocol).

TCP

Es un protocolo orientado a la conexión, lo que significa que se establece una conexión segura y confiale antes de que transmitan los datos. Los puertos que están asociados con TCP garantizan la entrega de forma correcta y en orden de los datos.

  • Por ejemplo: El puerto 443 se usa de forma común para la conexiones HTTPS de forma segura.

UDP

Este a difencia de TCP, es un protocolo sin conexión, lo que da a entender que puede envíar datos sin haber establecido una conexión previa, lo que da como resultado una transmición más rápida, pero menos fiable. Los puertos UDP se usan comúnmente para los juegos en línea o streaming, en donde es muy crucial la velocidad.

  • Por ejemplo: El puerto 53 se usa para las consultas DNS (Domain Name System) utilizando UDP.

Protocolos de Red

HTTP

HTTP(Hyper Text Transfer Protocol) Es el protocolo base de la World Wide Web. Se utiliza para transmitir datos, principalmente páginas web (documentos HTML), entre un navegador (cliente) y un servidor web.

  • El puerto que utiliza es el TCP 80.

HTTPS

HTTPS(Hyper Text Transfer Protocol Secure) Es una versión segura de HTTP que utiliza cifrado SSL/TLS para proteger la comunicación entre el cliente y el servidor.

  • El puerto que utiliza es el TCP 443.

FTP

FTP(File Transfer Protocol) Es un protocolo utilizado para transferir archivos entre un cliente y un servidor en una red. Permite la carga y descarga de archivos.

  • Los puertos que utiliza son el TCP 20 y 21.

FTPS

FTPS(File Transfer Protocol Secure) Es una versión segura de FTP que utiliza cifrado SSL/TLS para proteger la transferencia de archivos entre el cliente y el servidor.

  • Los puertos que utiliza son el TCP 21 y 990.

SFTP

SFTP(Secure File Transfer Protocol) Es un protocolo de transferencia de archivos que funciona sobre el protocolo SSH (Secure Shell). Proporciona una forma segura de transferir archivos entre un cliente y un servidor.

  • El puerto que utiliza es el TCP 22.

SMTP

SMTP(Simple Mail Transfer Protocol) Es un protocolo utilizado para enviar correos electrónicos entre servidores de correo.

  • El puerto que utiliza es el TCP 25 (tradicionalmente)
  • TCP 857 (moderno, con cifrado STARTTLS)
  • TCP 465 (antiguo, SMTPS)

POP3

POP3(Post Office Protocol version 3) Es un protocolo utilizado por los clientes de correo electrónico para recuperar correos electrónicos, generalmente descarga los correos del servidor al dispositivo local y los elimina del servidor (aunque puede configurarse para no hacerlo).

  • Utiliza el puerto TCP 110 (no cifrado)
  • Utiliza el puerto TCP 995 (POP3S, cifrado)

IMAP

IMAP(Internet Message Access Protocol) Es un protocolo utilizado por los clientes de correo electrónico para acceder y gestionar correos electrónicos almacenados en un servidor de correo. A diferencia de POP3, IMAP permite a los usuarios ver y organizar sus correos electrónicos directamente en el servidor sin necesidad de descargarlos.

  • Utiliza el puerto TCP 143 (no cifrado)
  • Utiliza el puerto TCP 993 (IMAPS, cifrado)

MIME

MIME(Multipurpose Internet Mail Extensions) No es un protocolo de red en sí mismo, sino un estándar que extiende el formato de los correos electrónicos para permitir la inclusión de contenido multimedia, como imágenes, audio, video y archivos adjuntos. MIME define cómo se codifica y estructura este contenido dentro de los correos electrónicos.

IRC

IRC(Internet Relay Chat) Es un protocolo de comunicación en tiempo real que permite a los usuarios participar en conversaciones grupales o privadas a través de canales de chat. Fue uno de los primeros protocolos de chat en línea y sigue siendo utilizado por comunidades en línea.

  • Utiliza el puerto TCP 6667 (no cifrado)
  • Utiliza el puerto TCP 6697 (IRC sobre SSL/TLS, cifrado)

XMPP

XMPP(Extensible Messaging and Presence Protocol) Es un protocolo de comunicación en tiempo real utilizado para mensajería instantánea, presencia y colaboración en línea. Es un protocolo abierto y extensible que permite a los usuarios enviar mensajes de texto, compartir archivos y participar en conferencias de chat.

  • Utiliza el puerto TCP 5222 (conexión cliente-servidor)

BitTorrent

BitTorrent Es un protocolo de intercambio de archivos peer-to-peer (P2P) que permite a los usuarios compartir y descargar archivos grandes de manera eficiente. En lugar de descargar un archivo desde un servidor centralizado, los usuarios descargan fragmentos del archivo desde múltiples fuentes (otros usuarios) simultáneamente.

  • Utliza el puerto TCP 6881-6889 (rango tradicional), aunque los clientes actuales usan puertos aleatorios de un rango más alto.

FastTrack

FastTrack Es un protocolo de intercambio de archivos P2P que fue popularizado por aplicaciones como Kazaa y Grokster. Permite a los usuarios compartir y descargar archivos directamente entre ellos sin necesidad de un servidor centralizado.

  • Utiliza el puerto TCP 1214

MTProto

MTProto Es el protocolo de comunicación utilizado por la aplicación de mensajería Telegram. Está diseñado para ser rápido, seguro y eficiente, permitiendo a los usuarios enviar mensajes, archivos y realizar llamadas de voz y video.

  • No tiene un puerto en específico, pero conmunmente usa el TCP 443 para camuflarse como tráfico web, puede usar el 80, 5222 o 25.
